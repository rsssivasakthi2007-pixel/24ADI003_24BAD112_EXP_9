In Scenario 1 (User-Based Collaborative Filtering), the system recommends movies to a user based on the preferences of other users with similar tastes. The dataset is first loaded and preprocessed by handling missing values and constructing a User-Item matrix. Cosine similarity is used to compute the similarity between users, and the top-N similar users are identified. Ratings for unseen movies are predicted using neighbors’ preferences, and personalized movie recommendations are generated. The model is evaluated using RMSE and MAE to measure prediction accuracy. Visualizations such as heatmaps of the User-Item matrix and similarity matrix help in understanding user relationships and sparsity patterns.

In Scenario 2 (Item-Based Collaborative Filtering), recommendations are generated by identifying items (movies) that are similar to those already liked by the user. An Item-User matrix is created, and similarity between items is computed using Cosine similarity or Pearson correlation. Based on a user’s interaction history, the system recommends top similar items. The approach is compared with user-based filtering in terms of accuracy, scalability, and recommendation quality. Evaluation metrics such as RMSE and Precision@K are used to assess performance. Visualizations include item similarity heatmaps, graphs of top similar items, and comparison charts between both methods.
